Введение

Vispons - это сервис для управления подписками и заданиями в Telegram ботах. API позволяет интегрировать функциональность проверки подписок и управления заданиями в вашего бота.

Для использования API вам необходимо зарегистрировать своего бота в панели управления Vispons и получить токен доступа.

Аутентификация

Все запросы к API Vispons должны содержать токен аутентификации в заголовках:

Authorization: Bearer YOUR_ACCESS_TOKEN

Замените YOUR_ACCESS_TOKEN на ваш действительный токен, полученный при регистрации бота.

Метод request_op

POST /api/request_op

Основной метод для проверки статуса подписки пользователя и получения списка ссылок для подписки.

Параметры запроса

Метод request_op принимает следующие параметры:

Параметр Тип Обязательный Описание
user_id Integer Да Уникальный идентификатор пользователя Telegram
chat_id Integer Да Идентификатор чата с пользователем
action String Нет Тип действия при запросе:
"subscribe" (по умолчанию): Подобранный список ссылок закрепляется за пользователем на время, указанное в настройках бота (параметр "Время сброса привязки"). При повторном запросе в течение этого времени будет проверяться подписка на тот же список.
"newtask": Подобранный список ссылок не закрепляется. При каждом запросе может быть выдан новый список. Актуально для разделов "Задания".
first_name String Нет* Имя пользователя. Обязательно если бот добавлен в сервис без токена.
language_code String Нет* Код языка пользователя. Обязательно если бот добавлен в сервис без токена.
is_premium Boolean Нет* Статус Premium подписки пользователя. Обязательно если бот добавлен в сервис без токена.
Примечание:

Если вы добавили свой бот в сервис БЕЗ токена, не забудьте передать дополнительные параметры: first_name, language_code, is_premium.

Получить их можно следующим образом:

message.from_user.first_name
message.from_user.language_code
message.from_user.is_premium

Обработка ответа

Метод возвращает объект с двумя полями: status и code.

Статус Код Описание
ok 200 Пользователь подписан на все необходимые каналы
ok 201 Пользователь только что подписался на каналы
wait 202 Ожидание подписки пользователя на каналы
error 400 Неверный запрос или отсутствуют обязательные параметры
error 401 Неверный или отсутствующий токен аутентификации
error 500 Внутренняя ошибка сервера

Callback обработка

Для обработки callback данных от кнопок подписки необходимо реализовать в вашем боте обработчик callback запросов.

@dp.callback_query(F.data.startwith("vispons"))
async def vispons_callback_query(call: types.CallbackQuery):
    user_id = call.from_user.id
    chat_id = call.message.chat.id
    if call.data == "vispons-op":
        status, code = await request_op(user_id, chat_id)
        if status == 'ok' and code == 200: #Если статус ok и code 200 (совокупность параметров означает, что пользователь подписан)
            #Выдать вознаграждение, если еще не было выдано
            await bot.send_message(chat_id, 'Спасибо за подписку👍')
        elif status == 'ok':
            #Пропустить пользователя дальше в своем боте
            await bot.send_message(chat_id, 'Спасибо за подписку👍')
        else:
            return
    elif call.data.startswith("vispons_gender_"):
        gender = call.data.split("_")[2]
        status, code = await request_op(user_id, chat_id, gender)
        if status == 'ok' and code == 200: #Если статус ok и code 200 (совокупность параметров означает, что пользователь подписан)
            #Выдать вознаграждение, если еще не было выдано
            await bot.send_message(chat_id, 'Спасибо за подписку👍')
        elif status == 'ok':
            #Пропустить пользователя дальше в своем боте
            await bot.send_message(chat_id, 'Спасибо за подписку👍')
        else:
            return
Примечание:

Если вы добавили свой бот в сервис БЕЗ токена, не забудьте передать дополнительные параметры в функцию request_op: first_name, language_code, is_premium.
